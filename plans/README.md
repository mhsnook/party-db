## Dependency notes

- **001 first**: 002/003/004 add integration tests and assume 001's typechecked
  harness and (optionally) its shared `connect` helper.
- **005 must not land before 003**: 005 turns on oplog retention by default, which
  makes the stale-cursor → fresh-snapshot path routine; before 003 that path
  corrupts reconnecting clients (duplicate-key throw, ghost rows).
- **006 after 002**: 002 removes a class of spurious throws from the catch that
  006 narrows, keeping 006's tests clean.
- **004 after 003**: 004's test reasoning leans on 003's `reset` marker semantics.
- **013 after 011**: both extend the `PartyCollection` type; 011 sets the
  `<T, Name>` shape 013 threads through. 013 has a mandatory human-review gate
  (its Step 3) before any implementation.
- 007, 008, 009, 010, 011 are independent of each other and of 001–006; they can
  run in parallel lanes. An actual npm publish should wait until at least
  002–006 + 008–010 are DONE.
- **014 comes after the correctness core**: it consumes 001's harness, 002's
  result-cursor discipline, and 006's error split; **003 and 004 are hard
  dependencies** — on D1 the `reset` snapshot is the *only* reconnect path (no
  oplog, no `?since` delta), and snapshot consistency assumes connects are
  serialized with writes, which D1's network round-trips make a real race
  rather than a microtask one.

## Findings considered and rejected

(So nobody re-audits these.)

- **PersistenceAdapter single-implementation "premature abstraction"** — the
  seam's async contract and the write-queue exist specifically for the documented
  D1 target (`docs/architecture.md` §4, roadmap); earning its keep per stated intent.
- **Example-app scaffolding triplication** (identical schemas/migrations/tsconfigs
  across `example*/`) — rejected by the maintainer 2026-07-08: examples/cookbooks
  are showcases, not maintained apps; cross-example reuse is a non-goal. Elide
  boilerplate in docs instead (`// ... same setup as …`).
- **Snapshot chunking/pagination** (whole-table snapshot in one WS frame,
  `sqlite-adapter.ts:197-201`) — deferred, not planned: real but premature at v0
  room sizes; revisit when D1 lands or a room-size complaint arrives. The
  retention default (plan 005) bounds the *oplog*, not snapshots.
- **`constraintOf` regex captures "NULL" for NOT-NULL violations** — cosmetic,
  best-effort field by design; not worth a plan.
- **Demo password printed on the example page** — intentional demo design
  (`example-react-rdbms/src/server.ts:6-8`); only the *echo in the 401 body* and
  the compare pattern were actionable (plan 010).
- **Dependency currency** — devDeps current (`vitest 4`, `typescript ^5.7`,
  pool `0.16.20`); no migration warranted. `pnpm audit` clean at planning time.
- **Auth default-open when no `authorize` is configured** — documented v0 posture
  (`docs/architecture.md` §10: the library validates nothing about your auth);
  not a finding.
